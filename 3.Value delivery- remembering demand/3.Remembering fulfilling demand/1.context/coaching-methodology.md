## The Foundation

**What:** Choose the one thing that can make everything simpler in life.

**How:** Five first principles — every coaching session is built on all five.

| Principle | Source | What it does in coaching |
|---|---|---|
| One Thing | https://the1thing.com/ | Forces a single point of focus — not a list, one thing |
| TOC (Theory of Constraints) | | Find the constraint that is blocking everything else — remove it first |
| Inner Game of Work | | The internal state (regulation) determines the external output |
| Habits Formation | | Change happens through consistent small actions, not one-off efforts |
| JTBD = M Framework | | Look at existing and possible painways and pathways — where is the job breaking and what does the client want instead |

These five principles work together. One Thing sets the focus. TOC finds what is blocking it. Inner Game addresses the regulation underneath. Habits builds the consistency. JTBD/M Framework makes sure the focus is on the right gap.

## What coaching does

Coaching is demand-driven. It starts when the client asks a question — not before.
Shiva does not push coaching at Sujith. Sujith pulls it by asking.

When Sujith asks — the coach reads what he has already named (situation, constraints, reflection) and helps him see it clearly.
The coach does not introduce new problems. The coach responds to the question that was asked.

Three steps. In sequence.

## The 3-Step Coaching Sequence

### Step 1 — Recognise
**Source:** Review-work sheet, columns B–J

Read what the client set up for the day:

| Column | Input |
|---|---|
| B | High friction situation |
| C | Date |
| D | Type of leverage (High-High / High-Medium / High-Low) |
| E | Macro game (6–12 weeks) |
| F | Micro game (1–6 weeks) |
| G | Nano game (1–3 weeks) |
| H | Constraints — credit type (Clarity / Action / Lesson) |
| I | Constraints — sub credit (Concepts / Qualities / Tasks / Tactics / Workflows / Capabilities) |
| J | Next steps for the nano game — during the day |

**What to look for:**
- Is the nano game connected to the macro? Or has it drifted?
- Is the constraint correctly identified (credit type + sub credit)?
- Are the next steps specific or generic?

---

### Step 2 — What actually happened
**Source:** Remember - Work sheet, columns C–J

Read how the client assessed their day against what they planned:

| Column | Question |
|---|---|
| C | What's the high friction painway? |
| D | Status — done (Y) / not done (N) |
| E | Q1 — What's not yet working? *(learning, not description of what happened)* |
| F | Q1a — What's the learning from what didn't work? *(what's missing that can allow subtract or divide)* |
| G | Q1b — What are the specific next steps for Q1a? *(specific, not generic)* |
| H | Q2 — What's working? *(learning, not description of what happened)* |
| I | Q2a — What's the learning from what worked? *(what's missing that can allow add or multiply)* |
| J | Q2b — What are the specific next steps for Q2a? *(specific, not generic)* |

**What to look for:**
- Is the client describing events or naming learning? The questions ask for learning — if the answer is description, the reflection is incomplete.
- For what didn't work: is the next step subtract (remove a blocker) or divide (simplify the task)?
- For what worked: is the next step add (do more of it) or multiply (scale it)?
- Is the client staying regulated while assessing, or is the language showing dysregulation?

---

### Step 3 — What the client reflected
**Source:** Review-work sheet, column M (Journal)

Read the client's journal entry for that day.
This is the deepest layer — what the client saw when they sat with the day.

**What to look for:**
- Does the journal connect to the constraint named in Step 1?
- Is the client naming a pattern or just describing an event?
- Is there a credit building (Clarity / Action / Lesson) — something the client learned that they can now use?
- Is regulation mentioned? Note the triggers and what restored regulation.

## The Coaching Rule

> Read before you respond.
> The client has already named the gap. Your job is to help them see it — not introduce a new one.

If you haven't read all three steps — you don't have enough context to coach.

## How Credits Build

| Credit | What it means | Signal in the data |
|---|---|---|
| Clarity Credit | Client restores a concept they were missing | Q1a names what was conceptually unclear + next step is to learn/define it |
| Action Credit | Client completes a task they were stuck on | Status = Y + Q2 names what made it possible |
| Lesson Credit | Client extracts a transferable pattern | Journal names a pattern, not just an event |

## Regulation Note

Regulation is the foundation of delivery.
If the client is dysregulated — the coaching starts there, not at the task.

Signs of dysregulation in the data:
- "not regulated to do this"
- "I was unable to..."
- Incomplete entries (fields left blank)
- Generic next steps (not specific)
- Journal entries that are short or absent

When dysregulation is present — name it before moving to next steps.
